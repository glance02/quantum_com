# 第三题技术说明

## 问题分析

本任务基于 Diabetic Retinopathy Debrecen 表格数据完成二分类。约束的重点有三点：

1. 两个模型必须共用完全一致的经典预处理流程。
2. 分类核心必须来自含可训练参数的量子变分线路。
3. 轻量模型需要在量子比特数和量子参数数上都小于基准模型。

## 方案概述

代码实现包含两个模型：

| 指标 | 基准模型 | 轻量化模型 |
| --- | ---: | ---: |
| 量子比特数 | 8 | 4 |
| 量子线路可训练参数 | 80 | 24 |
| 变分层数 | 3 | 2 |
| 量子测量输出维度 | 16 | 8 |

两者使用相同预处理：

1. 基于训练集统计量做 1%/99% 分位数截尾。
2. 对非负特征做 `log1p` 初等变换。
3. 使用中位数和四分位距做 Robust Scaling。
4. 将结果截断到 `[-3, 3]`，并线性映射到 `[-pi, pi]` 作为量子旋转角。

## 量子模型结构

每个模型由角度编码、数据再上传、可训练旋转、环形纠缠和 Pauli 期望测量组成。

每层结构：

1. 将经典特征编码到 `RY` / `RZ` 旋转门。
2. 每个量子比特施加 `RY(theta) -> RZ(theta) -> RX(theta)`，其中 `theta` 为可训练量子参数。
3. 使用环形 `CNOT(q, q+1)` 纠缠，奇偶层反向纠缠以增强表达能力。
4. 测量 `Z_i` 与相邻 `Z_i Z_{i+1}` 期望值。

基准模型在 3 层变分线路之后额外加入 8 个最终 `RY` 可训练旋转门，使量子参数数达到题目允许上限 80。

测量结果后接一层线性读出。基准模型经典读出参数为 `16 + 1 = 17`，轻量模型为 `8 + 1 = 9`，均低于题目限制的 50。

## 训练方式

`train.py` 只读取 `data/train.csv`，并从训练集中按类别划分验证集。量子参数使用 SPSA 方式优化，读出层在当前量子特征上拟合二分类交叉熵。

训练完成后保存两个独立模型文件：

```text
task3/artifacts/baseline_model.npz
task3/artifacts/lightweight_model.npz
```

同时额外保留一个合并文件 `task3/artifacts/qml_artifacts.npz` 作为兼容备份。`test.py` 不依赖训练流程，优先加载两个独立模型文件，并使用 `pyvqnet` 的 `QuantumLayerV3` 和 `pyqpanda3` 构造量子线路完成评估。

## 实验结果

本机运行：

```bash
python train.py
python test.py
```

一次运行示例输出：

```text
28.96,0.57,0.58,8,80,4,24
```

字段含义：

```text
Score,Acc_B,Acc_L,Q_B,P_B,Q_L,P_L
```

## 压缩分析

量子比特压缩率：

```text
(8 - 4) / 8 = 0.50
```

参数压缩率：

```text
(80 - 24) / 80 = 0.70
```

综合压缩比：

```text
0.5 * (0.50 + 0.70) = 0.60
```

轻量模型以较少量子比特和参数量保持了接近基准模型的测试准确率。
