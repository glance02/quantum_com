# 第三题 量子机器学习方向

满分 60 分

## 一、背景介绍

糖尿病视网膜病变（Diabetic Retinopathy, DR）是糖尿病引发的常见并发症，也是成年人失明的主要原因之一。早期检测和诊断对于预防视力丧失至关重要。本赛题基于 Diabetic Retinopathy Debrecen 数据集，要求参赛者使用量子机器学习方法完成糖尿病视网膜病变的二分类任务，探索量子线路的轻量化设计，在有限量子资源下实现性能与效率的平衡。

## 二、赛题设置

选手需要基于量子机器学习方法完成糖尿病视网膜病变二分类任务，鼓励选手研究**结合量子计算解决机器学习分类任务**。鼓励选手研究如何将量子模型进行压缩：首先设计一个中等规模的基准量子线路，然后在此基础上运用模型压缩技术，得到一个规模更小、但性能损失不大的轻量化量子线路。

### （一）模型提交要求

每位选手需要提交两个量子机器学习模型：

1. 基准模型（Baseline Model）：设计一个中等规模的量子线路以在测试集上取得较优性能：**量子比特 = 8，量子线路中的可训练参数数量在 [60,80]**。

2. 轻量化模型（Lightweight Model）：设计一个更小规模的量子线路以在测试集上取得较优性能：轻量化模型量子比特数 < 基准模型量子比特数，轻量化模型量子线路中的可训练参数 < 基准模型量子线路中的可训练参数数量。

### （二）训练与评估数据要求

两个模型都必须仅在题目给出的 `train.csv` 上训练，最终在 `test.csv` 上评估性能，可以在 `train.csv` 划分验证集（详见“三、数据集”部分）。

### （三）模型结构限制

最终提交的基准模型与轻量化模型中，除量子变分线路及其测量外，每个模型仅允许包含：

- 至多两层全连接层（其权重 + 偏置参数量总和不超过 50）；
- 至多两层固定的、不含可训练参数的激活函数。

最终模型本身不得包含任何其他经典机器学习模型参与训练或推理。

### （四）允许的数据处理（无标签、无训练参数）

| 处理类型 | 允许的具体手段 |
| --- | --- |
| 缺失值处理 | 常数填充（如 0、-1、`"missing"`）<br>均值、中位数、众数填充<br>前向/后向填充（时序） |
| 异常值处理 | 分位数截尾（如 1% / 99%）<br>IQR 截尾<br>固定阈值截断<br>中位数绝对偏差（MAD）截尾 |
| 缩放与标准化（仅基于简单统计量） | Z-score 标准化（均值 & 标准差）<br>Min-Max 归一化<br>MaxAbs 缩放<br>Robust Scaling（中位数 & 四分位距） |
| 初等函数逐元素变换 | log / exp / sqrt / square / reciprocal<br>sigmoid / tanh / sin 等三角函数（固定形式，无学习参数）<br>二值化（固定阈值） |
| 离散化 / 分箱（无监督） | 等宽分箱<br>等频分箱<br>基于固定区间的分箱 |
| 正交 / 线性代数变换（固定运算或仅基于数据统计量） | PCA / TruncatedSVD（无标签、无训练参数）<br>ZCA 白化<br>离散傅里叶变换（DFT）/ 快速傅里叶变换（FFT）<br>离散余弦变换（DCT） |
| 池化与累加 | 最大/平均/求和池化<br>累加和（cumsum）、累乘积<br>差分（diff） |

### （五）不允许的数据处理

| 处理类型 | 禁止的具体手段 |
| --- | --- |
| 需要从数据估计参数的无监督变换 | Box-Cox 变换（需估计 λ）<br>Yeo-Johnson 变换（需估计 λ） |

| 监督式特征选择或构造 | 基于互信息、卡方检验、ANOVA F 值的特征选择<br>基于树模型特征重要性的选择（如 Random Forest、XGBoost 重要性）<br>递归特征消除（RFE）<br>目标编码 / 均值编码（用目标变量均值替换类别）<br>WOE（证据权重）编码<br>Leave-one-out 编码、CatBoost 编码等 |

### （六）预处理与特征工程流程一致性

两个模型**必须使用完全一致的经典数据预处理与特征工程流程**，最终输入两个模型的经典数据特征维度**必须完全一致**（例如均为 N 维，N 由选手自行设置）。将经典数据编码为量子态的量子编码线路属于模型结构的一部分。

### （七）分类决策要求

最终提交的量子机器学习模型中，分类决策的核心必须由量子线路及其可训练参数完成，经典神经网络仅起辅助作用（如至多两层全连接层）。

**禁止以下情况**：仅将量子线路作为固定的、无可训练参数的特征编码器或特征映射（即量子线路只进行数据编码和期望值测量，不包含参数化量子门），后接经典机器学习模型进行分类。

### （八）训练与评估禁止行为

针对最终提交模型的训练与评估，禁止以下行为：

1. 使用任何本次比赛提供的数据（`train.csv`、`test.csv`）以外的外部数据；

2. 使用在任何外部数据上预训练过的模型权重（包括经典模型和量子模型）；

3. 将测试集（`test.csv`）以任何形式用于训练、验证或超参数选择（仅允许最终评估时使用）。

4. 允许在从基准模型到轻量模型的压缩过程中使用辅助方法，辅助方法严禁改变输入量子模型的数据特征（**须满足赛题设置第（四）（五）点**），辅助过程中使用的任何模型或数据必须完全来源于本次比赛的训练集（`train.csv`），不得引入外部数据、外部预训练权重或测试集信息。最终模型的训练不得依赖于测试集或外部数据。

### （九）交付物说明

1. 使用 `pyvqnet>=2.17.3` 或 `pyqpanda3>=0.3.2` 进行两个量子机器学习模型的训练与评估。可采用其他常见 Python 包进行其他功能开发，但必须遵守本赛题前述规则的限制。

2. 选手需要提供 `requirements.txt` 为运行代码所需 Python 软件列表（使用 `pip freeze > requirements.txt`）。

3. 选手需要提交可以运行的代码文件 `train.py`（用于完成两个模型的训练任务或从基线模型到轻量化模型的压缩与优化设计任务）、`test.py`（不依赖训练流程，直接完成两个模型在测试集上的评估任务）；`test.py` 在 4 核 CPU 和 16GB 内存个人计算机上的运行时长不超过 10 分钟。

4. 提交说明：

   另提交一个技术说明文档，为 `.pdf` 格式，命名为：`origincup1.3.pdf`；

   将 `test.py`、`requirements.txt`、`train.py`（不允许对文件名进行修改）打包并命名为：`origincup1.3`，压缩成 `.zip` 格式文件上传。

## 四、评分标准

### （一）客观分，满分 50 分

评委会运行 `test.py` 后得到测试集上的运行结果，需要输出以下指标：

$Score$, $Acc_B$, $Acc_L$, $Q_B$, $P_B$, $Q_L$, $P_L$（如 `37.24,0.9,0.7,8,80,4,50`），不同数值之间用逗号间隔。其中 $Score$ 作为客观分分数。

$$
Score = 22 \times Acc_B + 22 \times Acc_L + 6 \times CompressionRatio \times \min\left(\frac{Acc_L}{Acc_B}, 1\right)
$$

其中：

$$
CompressionRatio = 0.5 \times \left[\frac{Q_B - Q_L}{Q_B} + \frac{P_B - P_L}{P_B}\right]
$$

1. $Acc_B$：基准模型测试准确率（0~1）

2. $Acc_L$：轻量模型测试准确率（0~1）

3. $Q_B$、$P_B$：基准模型的量子比特数、量子线路中待训练参数个数

4. $Q_L$、$P_L$：轻量模型的量子比特数、量子线路中待训练参数个数

数值四舍五入，保留小数点后两位。

其他注意事项：

1. 运行 `test.py` 无法正确输出以上指标，客观分记为 0 分；

2. `train.py` 需要包含如何获取其基准模型、轻量模型的过程，并可以正常运行，否则客观分记为 0 分；

3. **若选手在实现算法的过程中违反赛题设置中的（一）至（四）、（六）至（九）的任一项，客观分记为 0 分。**

### （二）主观分，满分 10 分

提交一个技术文档，内容应包括：

1. 问题分析：简述对赛题任务和难点的理解分析；

2. 方案概述：对现有可行方法的简述，以及所选方案的描述；

3. 原理解释：基准模型和轻量化模型的设计原理说明，压缩优化方法说明；

4. 实验评估：结合实验结果简述方案的性能表现；

5. 复杂度分析：对所使用两种量子模型的性能以及资源复杂度（量子比特数、参数数量）进行对比分析。例如：

| 指标 | 基准模型 | 轻量化模型 |
| --- | --- | --- |
| 量子比特数 | $Q_B$ | $Q_L$ |
| 可训练参数总数 | $P_B$ | $P_L$ |
| 测试集准确率 | $Acc_B$ | $Acc_L$ |

量子比特压缩率：

$$
\frac{Q_B - Q_L}{Q_B}
$$

参数压缩率：

$$
\frac{P_B - P_L}{P_B}
$$

综合压缩比：

$$
\frac{1}{2}\left(\frac{Q_B - Q_L}{Q_B} + \frac{P_B - P_L}{P_B}\right)
$$

主观分评分为整数，评分会综合考虑以下维度：

1. 代码的合规性、可运行性；

2. 文档描述与实际提交代码的一致性；

3. 所使用量子机器学习算法的创新性与合规性；

4. 量子线路压缩算法的有效性；

5. 文档的逻辑条理性、图表规范性、结论分析的深度；

6. 未在上述赛题设置第（四）点中明确列出的经典预处理操作，一律默认视为违规。即使某方法未在赛题设置第（五）点中明确写出，评审认定该方法实质上“通过复杂经典手段预先提取了判别特征”或“显著削弱了量子计算环节的必要性”，主观评分将进行在“压缩方法创新性与有效性”与“算法的高效性与合规性”方面扣分。
